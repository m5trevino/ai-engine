Feature Evolution: Auth & Forensics

START: Open System, Global Logs
No authentication. All strike logs (success/fail) written to global files like success_strikes.log. No user or app separation.
↓
UPGRADE 1: Identity Partitioning
Introduced an optional identity parameter. Forensic logs were partitioned into separate directories: vault/{identity}/. This created clean audit trails for different users or missions.
↓
UPGRADE 2: WebUI Forensic Explorer
Built a secure, browser-based interface for auditing the partitioned vaults. Added API endpoints like /v1/vault/list to serve log data to the frontend.
↓
FINAL: "Syndicate" Identity-First Architecture
Made authentication mandatory. Unauthenticated strikes are rejected. All operations require a pk_ secret in the Authorization: Bearer header, scoping every action to a specific app profile with its own isolated vault and custom configs.
