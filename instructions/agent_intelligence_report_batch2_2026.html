25 more frameworks: platforms, memory layers, coding agents, and specialists

Cursor
The AI-native code editor — flow state for devs
Coding agent
Famous for
  • +Most broadly adopted AI coding tool among individual devs and small teams in 2026
  • +Autocomplete + chat inside the editor — never breaks flow to switch tools
  • +"Stays out of the way" — fast, quiet, reliable on small-to-medium tasks
  • +Multi-file context awareness baked into every suggestion
Struggles with
  • –Large, complex refactors — looping behavior and incomplete repo-wide changes are reported
  • –Pricing changes have generated significant backlash ("pay more, get less")
  • –Weaker than Claude Code on deep architectural reasoning and unfamiliar codebases
GitHub Copilot
Microsoft's IDE-embedded pair programmer
Coding agent
Famous for
  • +Trained on billions of public repos — suggests widely adopted patterns and best practices
  • +Native IDE integration across VS Code, JetBrains, and more — zero context switching
  • +Agent Mode (2025): orchestrates multi-file changes, tests, and iteration autonomously
  • +Mission Control (2026): runs multiple agent tasks in parallel as a team
  • +GitHub PR-aware — reads issues, discussions, and repo context for agentic tasks
Struggles with
  • –Web-based agent has 90+ second cold boot times — "stop-n-go" workflow is frustrating
  • –Sandboxed environment: restricted internet, limited repo permissions, all commits need human review
  • –Doesn't learn from private repositories — suggestions not personalized to your codebase style
Claude Code
Terminal-native agent for hard engineering problems
Coding agent
Famous for
  • +80.8% SWE-bench Verified — #1 ranked AI coding agent in 2026
  • +Universally described as the escalation path when other tools fail on hard problems
  • +Deepest reasoning on unfamiliar codebases, architectural changes, and subtle bugs
  • +Terminal-native — reads, writes, refactors, and debugs across entire repositories
  • +Trusted for design-level changes where correctness matters more than speed
Struggles with
  • –Expensive — usage-based API billing surprises teams at scale
  • –Line-by-line autocomplete is slower than Cursor/Copilot — not built for flow state
  • –No persistent IDE integration — requires terminal context management from user
Mem0
Universal persistent memory layer for agents
Memory layer
Famous for
  • +26% accuracy boost over OpenAI Memory baseline on LOCOMO benchmark
  • +91% faster responses and 90% lower token usage than full-context approaches
  • +Four memory types: episodic, semantic, procedural, and associative
  • +Actor-aware memory — tracks which agent said what in multi-agent systems
  • +Exclusive AWS Agent SDK memory provider; 19 vector store backends supported
Struggles with
  • –Requires an LLM to function — not fully self-contained
  • –Temporal reasoning is weaker than Zep's graph-based approach
  • –Cloud tier has SOC 2 compliance but self-hosted version puts all security on you
Letta
Formerly MemGPT — stateful agents with deep memory control
Memory layer
Famous for
  • +Procedural memory — agents can rewrite their own system instructions over time
  • +Tool-based memory architecture gives complete control over how memory works
  • +Long-term memory built into the agent runtime, not bolted on as a side layer
  • +Open-source and self-hostable — full data sovereignty
Struggles with
  • –Smaller community than Mem0; fewer production deployments documented
  • –MemGPT → Letta rebrand created confusion and migration friction
  • –No managed cloud option — self-hosting burden is entirely on the team
Flowise
Visual LangChain agent builder — now Workday-backed
Low-code
Famous for
  • +Best visual tool for wiring full LangChain-based agent stacks without code
  • +Chatflow (single agent) + Agentflow (multi-agent) cover both simple and complex cases
  • +Human-in-the-loop checkpoints built into visual flows natively
  • +Acquired by Workday (Aug 2025) — enterprise infrastructure and compliance now backing it
  • +30k+ GitHub stars; YC-backed origin with strong developer community
Struggles with
  • –Debugging visual flows is harder than debugging code — errors are harder to trace
  • –Performance limits on highly complex or high-volume scenarios
  • –Workday acquisition raises questions about future open-source direction for indie builders
Voiceflow
The Figma of conversational AI design
No-code
Famous for
  • +Best-in-class collaboration tool — designers, PMs, and CX teams all work in one canvas
  • +Voice AND chat in one platform — telephony channel is unique vs most competitors
  • +Fastest path from whiteboard to working prototype for conversation flows
  • +Knowledge base ingestion (FAQs, help docs) → live agent in minutes
  • +Multi-LLM flexibility without managing model infrastructure
Struggles with
  • –No self-hosting option — cloud-only with limited enterprise data residency controls
  • –Heavy backend logic requires external tools (devs add Flowise/n8n alongside it)
  • –Not suited for agentic task execution beyond conversation flows
Botpress
Developer-controlled open-source chatbot platform
No-code
Famous for
  • +Available on the most channels of any chatbot builder (10+ including WhatsApp, Teams, Telegram)
  • +Built-in live chat handoff + native Zendesk integration — real support workflows
  • +50+ direct app integrations with fine-grained LLM customization
  • +Self-hostable open-source core for full data control
  • +Best analytics of any chatbot builder in the segment
Struggles with
  • –Overkill for simple FAQ bots — Voiceflow is faster for basic use cases
  • –Self-hosting means your team owns server management, updates, and scaling
  • –Less intuitive UX than Voiceflow for non-technical teams
Lindy AI
Build AI employees in plain English
No-code
Famous for
  • +5,000+ integrations — broadest workflow connectivity of any no-code agent platform
  • +Natural language setup: describe your agent, Lindy builds it — no flow builder needed
  • +Best for non-technical teams automating communication, scheduling, and CRM tasks
  • +$50M raised, ~40k active users — strong product-market fit signal
Struggles with
  • –Credit-based billing surprises users — costs scale unpredictably at volume
  • –Limited developer control for custom logic or non-standard integrations
  • –Not suitable for complex stateful workflows that need checkpointing or rollback
Swarms Framework
Enterprise-grade swarm orchestration
Code-first
Famous for
  • +Most orchestration patterns of any framework: hierarchical, concurrent, sequential, graph, and swarm
  • +MAKER architecture: micro-agent voting + red-flagging for statistical agreement on each step
  • +Python AND Rust support — performance-critical deployments can run native speed
  • +Agent registry + marketplace for reusable enterprise swarm components
  • +Built-in high availability, observability, and legacy system integration
Struggles with
  • –Complexity is high — not for teams without serious multi-agent architecture experience
  • –Smaller community and fewer production war stories vs LangGraph or CrewAI
  • –Documentation does not yet match the speed of feature development
OpenAI Swarm
Educational lightweight multi-agent handoff framework
Code-first
Famous for
  • +Cleanest possible reference implementation of multi-agent handoff patterns
  • +Minimal abstraction — ideal for learning how coordination actually works under the hood
  • +Fast to prototype: 2–5 agent patterns with explicit handoffs in minutes
  • +OpenAI explicitly manages it as an educational framework — documentation is clear
Struggles with
  • –No persistence, no error recovery, no orchestration logic for production
  • –OpenAI explicitly says: do not deploy to production
  • –Superseded by OpenAI Agents SDK for anything beyond experiments — don't migrate to Swarm
Microsoft TaskWeaver
Code-first data analytics agent from Microsoft Research
Code-first
Famous for

  • +Container-mode execution for isolated, secure code runs by default
  • +Plugin system for encapsulating custom domain algorithms
  • +Built-in AgentOps integration for observability and monitoring

  • –Limited enterprise adoption outside Microsoft-adjacent data science teams
  • –Vision input (for Planner role) is still experimental as of early 2026
