Feature Evolution: The UI

START: Legacy Alpine.js UI
A 1,700+ line monolith described as "filler" and "Ghost UI." Prone to syntactic errors and routing collisions.
↓
PIVOT: High-Fidelity Vite/React Architecture
Shifted from static HTML/Alpine to a professional Multi-Page App (MPA) structure using Vite, then to a full Single-Page App (SPA) using React/TypeScript. New home: ui/.
↓
HARDENING: UI Unification & Root-Serve
Purged the legacy app/static directory. Refactored main.py to serve the single React build from the root (/) to eliminate prefixes and 404s.
↓
STABILIZATION: SPA Routing & API Link Fixed
Implemented a "catch-all" route in FastAPI to support client-side navigation. Removed hardcoded localhost URLs in api.ts for dynamic, production-ready API resolution.
↓
UPGRADE: "Sand Hill" Dashboard Overhaul
Upgraded typography to high-contrast fonts (Geist/Outfit). Implemented a collapsible right sidebar for advanced run settings, shifting to a Google AI Studio-style layout.
↓
FINAL: Dedicated Payload Striker Screen
Isolated all complex batch/campaign logic from the main chat dashboard into its own dedicated /striker screen to maintain architectural integrity.
